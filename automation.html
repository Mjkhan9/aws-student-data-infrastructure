IAM Automation Pipeline (Python + Boto3)

To eliminate manual IAM onboarding and reduce the risk of misconfiguration, the project includes a Python-based automation script that provisions users into a restricted group with read-only access to student data exports.

The script can run in a demo mode without AWS credentials (for safe demonstration) or live in an AWS environment when credentials are configured.

Automation Objectives

  • Standardize IAM identity creation for student data roles.
  • Guarantee least-privilege access aligned with policy.
  • Provide measurable improvement over manual onboarding.
  • Produce clear logs for faculty and audit review.

Core IAM Design

  • Group: StudentDataRestrictedAccess
  • Policy: S3 read-only access to reporting bucket
  • Users: Registrar, data specialists, reporting systems
  • Access Scope: Read-only access to curated exports, not raw databases

Python Workflow (iam_provisioner.py)

  1. Initialize IAM client (or demo mode if boto3/credentials unavailable).
  2. Validate that the StudentDataRestrictedAccess group exists.
  3. Attach the S3 read-only policy to the group if not already present.
  4. Iterate through a list of user identities and provision each one.
  5. Add each user to the restricted group.
  6. Print a summary of provisioning time and efficiency gains.
