IAM Automation Overview

Automated IAM provisioning workflow designed for the AWS Student Data Infrastructure project. The automation script ensures users are created securely, assigned correct permissions, enforced with MFA, and tagged for audit visibility.

IAM Provisioning Workflow

  1. Create IAM User with enforced MFA requirements
  2. Attach IAM Groups based on access type
  3. Tag User with role, email, and application ownership
  4. Create Login Profile with secure password rotation
  5. Record actions into CloudTrail & execution logs
