AWS VPC Architecture

The environment uses a fully isolated VPC architecture modeled after industry AWS best practices for sensitive student information systems. Each layer (Admin, Application, Database) is network-isolated, encrypted, and monitored.

VPC Overview
A 3-tier architecture with enforced security boundaries.

Security Controls
  • Bastion/VPN required for admin-tier access.
  • Application tier unreachable from the internet.
  • Database tier fully private with SG-to-SG permissions.
  • Network ACLs restrict lateral movement.
  • All data encrypted using AWS KMS (RDS, S3, snapshots).
  • CloudTrail logs IAM, RDS, SG, and VPC modifications.
Student Data Flow
  1. Admins access system via secure bastion or VPN.
  2. App tier processes student workflows (read/write, validation, audit).
  3. All sensitive data stored in encrypted RDS (PostgreSQL).
  4. Exports written to S3 with strict IAM access boundaries.
  5. CloudWatch + Flow Logs provide monitoring and alerting.
