AWS VPC Architecture

The network design follows a three-tier VPC model with dedicated segments for administration, application workloads, and the database layer. The goal is to protect student data while enabling controlled access for authorized systems and users.

Security Controls

  • Only the admin tier has controlled ingress from the internet (via bastion or VPN).
  • Application tier cannot be reached directly from the internet or student devices.
  • Database tier is fully private; only app-tier security groups can reach RDS ports.
  • Network ACLs restrict lateral movement between subnets.
  • Security groups favor SG-to-SG referencing over IP-based rules.
  • All data at rest in RDS and S3 is encrypted via AWS KMS.

Data Flow for Student Information

  1. Authorized staff connect to the admin tier using secure channels (VPN / bastion).
  2. Application services in the app tier process student-related workflows.
  3. Application services read/write student records to the RDS PostgreSQL instance in the DB tier.
  4. Reports or exports are written to an encrypted S3 bucket with strict IAM policies.
  5. CloudTrail and VPC flow logs capture access and network activity for audit.
