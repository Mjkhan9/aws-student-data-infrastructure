Running the Automation Script

  1. Install Python 3.x on your local machine.
  2. Optionally, install Boto3 with pip install boto3.
  3. Clone or download this repository.
  4. Open a terminal in the project directory.
  5. Run python iam_provisioner.py.

By default, the script runs in a safe demo mode that does not require AWS credentials. In a live environment, demo mode can be disabled and IAM changes will be applied directly to the configured AWS account.

Security & Compliance Considerations

  • Principle of least privilege applied to all IAM roles and groups.
  • Student data isolated to a private RDS subnet with no direct internet exposure.
  • All at-rest data protected using AWS KMS-managed or customer-managed keys.
  • CloudTrail enabled to capture IAM, RDS, and VPC configuration events.
  • Network-level separation between admin tools and application/data tiers.
