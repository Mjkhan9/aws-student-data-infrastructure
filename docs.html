Project Documentation Overview

This section provides full documentation for the AWS Student Data Infrastructure project, including the architecture summary, automation design, execution outputs, and academic review. The goal is to provide complete transparency into the project's planning, infrastructure decisions, and IAM implementation.

All documentation presented here is sourced directly from project files included in the GitHub repository.

Architecture Summary

The following architecture summary is based on the architecture_summary.md file:

Three-tier VPC architecture with public, private app, and private database subnets. NAT gateway for outbound-only internet access for private subnets. RDS PostgreSQL with encryption, private networking, and subnet group isolation. Security groups configured with least privilege and SG-to-SG rules. IAM roles separating read-only, admin, and automation functions. CloudTrail enabled for all regions with S3 log storage and validation.

Core Design Goals

  • Maintain full FERPA-aligned data isolation
  • Eliminate all public attack surfaces
  • Enforce IAM least privilege and MFA
  • Ensure full auditability via CloudTrail & VPC Flow Logs

IAM Automation Documentation

IAM automation is handled by the iam_provisioner.py script. It provisions users, attaches permissions, enforces MFA, and logs actions for auditing.

Key Features

  • Automatic user creation with tagging
  • Group-based permission assignment (role-access)
  • MFA enforcement through IAM deny policy
  • Optional access key generation
  • Execution results saved to execution_output.txt

Excerpt from iam_provisioner.py

iam.create_user(UserName=username) iam.add_user_to_group( GroupName=f"{role}-access", UserName=username ) iam.put_user_policy( UserName=username, PolicyName="MFAEnforcement", PolicyDocument="{ ... }" )
