Project Documentation Overview

Full documentation for the AWS Student Data Infrastructure project, including architecture summary, automation design, execution outputs, and technical specifications.

Architecture Summary

  • Three-tier VPC architecture with public, private app, and private database subnets
  • NAT gateway for outbound-only internet access from private tiers
  • RDS PostgreSQL with encryption and private networking
  • Security groups configured with least privilege and SG-to-SG references
  • CloudTrail enabled for all regions with log file validation
  • VPC Flow Logs capturing all accepted and rejected traffic

Core Design Goals

  • Maintain FERPA-aligned data isolation and access controls
  • Eliminate all public attack surfaces through network segmentation
  • Enforce IAM least privilege and multi-factor authentication
  • Ensure full auditability via CloudTrail and VPC Flow Logs
  • Implement defense-in-depth with multiple security layers
  • Enable high availability through multi-AZ deployment

Security Controls

  • Network Isolation: Database tier has no internet gateway route
  • Encryption: Customer-managed KMS keys for all data at rest
  • Access Control: Tag-based S3 access policies and IAM conditions
  • Monitoring: GuardDuty threat detection and CloudWatch alarms
  • Audit Logging: Complete audit trail of all API calls and network traffic

Technology Components

  • Networking: AWS VPC, Security Groups, NACLs, NAT Gateway, VPC Endpoints
  • Compute: EC2, ECS Fargate, Lambda, Application Load Balancer
  • Database: RDS PostgreSQL Multi-AZ with automated backups
  • Storage: S3 with SSE-KMS encryption and versioning
  • Security: IAM, KMS, CloudTrail, GuardDuty, AWS WAF
  • Monitoring: CloudWatch, VPC Flow Logs, CloudTrail
